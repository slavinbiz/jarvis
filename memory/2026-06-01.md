# Дневник 2026-06-01

## Тема сессии
Подключение Google Calendar через MCP

## Сделано

### Авторизация Google Calendar MCP
- Выяснили, что MCP инструменты calendar (list_events и др.) возвращали "The caller does not have permission"
- Причина 1: в credentials.json было ДВА токена с разными ключами — старый (5cc3ae4d874c4870, из calendar-auth.mjs) и правильный (579e90e4bdd6e384, сохранённый Claude Code)
- Причина 2: mcpOAuthClientConfig использовал старый ключ
- Решение: удалили старый токен, скопировали clientConfig на правильный ключ
- Токен проверен: работает напрямую с Google Calendar API (возвращает 4 календаря)
- MCP инструменты заработают на следующей сессии (каждый запуск читает credentials заново)

### Попытки OAuth через MCP authenticate
- mcp__google-calendar__authenticate + mcp__google-calendar__complete_authentication
- Проблема: OAuth state хранится в памяти Claude Code и сбрасывается между сообщениями
- state mismatch / "No OAuth flow is in progress" — известное ограничение HTTP MCP на VPS

## Решения

- Правильный ключ для Google Calendar MCP: `google-calendar|579e90e4bdd6e384` (SHA-256 от URL `/mcp/v1`)
- Старый ключ `google-calendar|5cc3ae4d874c4870` — был из calendar-auth.mjs, некорректный
- Для показа событий: прямой запрос к `www.googleapis.com/calendar/v3` с Bearer токеном
