### Проблемы и решения

**Проблема 1: npm не мог скачать пакет**
- Причина: в переменных окружения Windows застряли прокси от v2rayN (`socks5://127.0.0.1:10810`)
- Решение: очистить через `$env:HTTP_PROXY = ""` и `$env:HTTPS_PROXY = ""`, потом `[System.Environment]::SetEnvironmentVariable("HTTP_PROXY", $null, "User")`

**Проблема 2: Claude Code не подключался к Anthropic (заблокировано в РФ)**
- Причина: нужен VPN, но Habb — только браузерный
- Решение: Hysteria2 туннель на Windows через ярлык `Hysteria2-VPN-Tunnel` в автозагрузке

**Проблема 3: туннель не стартовал автоматически**
- Проверка: `Test-NetConnection 127.0.0.1 -Port 10810`
- Если `False` — запустить ярлык вручную: Win+R → `shell:startup` → двойной клик на Hysteria2-VPN-Tunnel

**Проблема 4: Claude Code не понимал SOCKS5 прокси**
- Переменные Windows имели `socks5://` — Claude Code не поддерживает
- Решение: явно задать HTTP прокси: `$env:HTTPS_PROXY = "http://127.0.0.1:10810"`

**Проблема 5: VS Code extension выдавала 403**
- Расширение Claude Code в боковой панели VS Code не работает через туннель
- Решение: использовать CLI в отдельном PowerShell окне

## Как запускать Claude Code на Windows (итоговая схема)

1. Убедиться что PowerShell с туннелем открыт (или запустить Hysteria2-VPN-Tunnel из автозагрузки)
2. Проверить: `Test-NetConnection 127.0.0.1 -Port 10810` → должно быть `True`
3. Открыть **новый** PowerShell (отдельное окно, не VS Code terminal)
4. Задать прокси:
   ```
   $env:HTTPS_PROXY = "http://127.0.0.1:10810"
   $env:HTTP_PROXY = "http://127.0.0.1:10810"
   ```
5. Перейти в папку проекта:
   ```
   cd "C:\Users\User\Documents\ИИ и прочее\вайбкодинг\Дмитрий Ледовских-курс\jarvis"
   ```
6. Запустить: `claude`
7. При предупреждении Bypass Permissions → выбрать **2 (Yes, I accept)**

### Автозапуск Claude Code — скрипт + ярлык на рабочем столе

Чтобы не вспоминать команды и не открывать вручную туннель — сделали автоматизацию:

- `scripts/start-claude.ps1` — проверяет туннель на 10810, если не отвечает — сам запускает ярлык Hysteria2-VPN-Tunnel из автозагрузки и ждёт до 10 сек, потом выставляет `$env:HTTPS_PROXY`/`HTTP_PROXY` и стартует `claude` в папке jarvis
- Поймали баг: Write-инструмент сохраняет `.ps1` в UTF-8 без BOM, а Windows PowerShell 5.1 без BOM читает кириллицу в пути и тексте неправильно (мусор вместо русских букв, скрипт не парсился). Решение: пересохранить файл как UTF-8 **с BOM** (`[System.Text.UTF8Encoding]::new($true)`) — для любых будущих `.ps1` с кириллицей делать так же
- Создан ярлык **«Claude Code»** на рабочем столе (`C:\Users\User\Desktop\Claude Code.lnk`) → запускает `powershell.exe -NoExit -ExecutionPolicy Bypass -File ...\start-claude.ps1`
- Execution Policy на машине — `RemoteSigned` для CurrentUser, локальные скрипты и так разрешены, `-ExecutionPolicy Bypass` в ярлыке на всякий случай
- Проверили запуск программно (Start-Process на .lnk, потом проверка дочерних процессов через Win32_Process) — дошло до старта `claude.exe`, значит туннель+прокси+запуск работают
- Обновили `knowledge/claude-code-windows.md` (добавили раздел «быстрый способ» через ярлык) и `PREFERENCES.md` (инструменты: Claude Code CLI в отдельном PowerShell)
- Закоммитили и запушили в `slavinbiz/jarvis` (коммит `6107407`)
