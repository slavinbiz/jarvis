- Разобрал баг «бот сигналит не туда» на реальном примере (HEMI/USDT, контр-сигнал SHORT недельная EMA): вход EMA14(W) 0.0062676, но стоп 0.0055998 оказался НИЖЕ входа — а для шорта стоп обязан быть выше. Причина: `nearest_stop_level()` в `ema_pullback.py` берёт хай/лой только из недавних внутридневных свечей, не сверяясь с положением входа. Когда вход — далёкий недельный откатный уровень, куда цена после пампа/дампа ещё не доходила, недавний хай/лой оказывается ближе к текущей цене, чем к входу, и стоп у шорта уезжает ниже входа (та же сторона, что и тейк) — по факту защита от движения против сделки не работает. Баг появился 20.07 при переходе стопа с недельной EMA на внутридневной хай/лой (см. запись сессии 1 выше) — переход тестировался только на сценариях, где цена уже рядом со входом
- Фикс: стоп теперь клампится относительно входа — `stop_level = max(stop_level, entry)` для шорта / `min(stop_level, entry)` для лонга — перед буфером 2%. Гарантирует стоп всегда по правильную сторону от входа. ### Решения
- Стоп через хай/лой считаем с буфером 2% (как раньше для EMA), не ровно по экстремуму
- Источник экстремума — внутридневные свечи (1h/30m), не недельные — раньше предлагали недельные, отклонено, т.к. работа внутри дня
- IIV-гейт не убирали совсем (защищает от шума на тонких парах) — добавили обходной путь только для сильных движений (2x порога)
