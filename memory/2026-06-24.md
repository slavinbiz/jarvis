# Дневник 2026-06-24

## Тема сессии
Деплой Lead Machine на VPS Beget Latvia (91.193.25.237)

## Сделано

### Доступ к серверу
- Прямой SSH с этой машины не проходил (нет ключа в authorized_keys, нет `expect`)
- Решение: PuTTY `plink`/`pscp` с паролем (`-pw`), уже установлен в системе — рабочий аналог expect для Windows
- Подключились под root после двух попыток пароля

### Подготовка сервера
- Диск был забит на 99% (153 МБ свободно) — почистили мёртвый кэш VS Code Tunnel (`/root/.vscode/cli/servers`, 2.9 ГБ, туннель даже не был запущен), apt-кэш, журналы. Освободили 3.3 ГБ
- Установили Docker (get.docker.com) — его не было

### Деплой кода
- Repo `lead-machine` приватный — git clone на сервере не прошёл (нет токена там). Перенесли тарболлом через pscp вместо этого
- Перенесли уже авторизованную Telethon-сессию (`data/sender_session.session`) — повторная авторизация не понадобилась
- Поправили `docker-compose.prod.yml`: app использует `network_mode: host`, поэтому db/redis нужны `ports: 127.0.0.1:5432/6379` — без этого app не достучался бы до localhost:5432/6379
- Добавили `DB_PASSWORD` в `.env` для подстановки `${DB_PASSWORD}` в compose
- Поймали баг: `Dockerfile` делает `COPY . .`, значит `.env` копируется в образ при сборке — любая правка .env требует пересборки
- Поймали баг в коде: aiogram 3.7.0 больше не принимает `parse_mode=` в `Bot()` — нужен `default=DefaultBotProperties(...)`. Исправили `app/bot/control.py`, закоммитили и запушили в github.com/slavinbiz/lead-machine
- Добавили `extra="ignore"` в `Settings.model_config` (app/config.py), чтобы лишний DB_PASSWORD не валил pydantic-валидацию

### Результат
- FastAPI + dispatch_loop запущены и работают
- Alembic-миграции прошли (initial schema)
- DB (postgres:16) и Redis — контейнеры здоровы
- **Не работает:** Control Bot — `TELEGRAM_BOT_TOKEN` в `.env` оказался плейсхолдером, не настоящим токеном. Падает `TelegramUnauthorizedError`

## Решения

- На этой Windows-машине для SSH с паролем — PuTTY plink/pscp, не expect (его нет)
- Приватные репо на сервер — тарболлом через pscp, не git clone (нет токена на сервере)
- Все детали и команды для повтора/дебага — knowledge/lead-machine-deploy.md
