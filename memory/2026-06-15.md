# 2026-06-15

## Сессия 1
Тема: починка VPN (3x-ui на TimeWeb 64.188.57.249) для браузера и Telegram на Windows через v2rayN

### Сделано
- Диагностировали: сервер сам нормально ходит в интернет и до Telegram (не блокировка локации/Германии)
- Проверили конфиг VLESS+Reality в v2rayN — полностью совпадает с сервером (UUID, pbk, sid, sni, fp, spx, flow, encryption)
- Нашли корень проблемы: `Test-NetConnection` с Windows до порта 21533 (и 2096) не проходил, до 22 — проходил. OS-файрвол (ufw) на сервере выключен, iptables всё разрешает — проблема была в **облачном файрволе TimeWeb**, отдельном от ОС, который по умолчанию открыт только на 22
- Поменяли инбаунд 3x-ui с порта 21533 на 443 (через sqlite3 в /etc/x-ui/x-ui.db, `update inbounds set port=443 where id=1`) + `systemctl restart x-ui`
- Вячеслав открыл порт 443 в панели TimeWeb (облачный файрвол)
- Проверили: `Test-NetConnection -Port 443` → True, `curl.exe -x socks5h://127.0.0.1:10808 https://api.ipify.org` → вернул IP сервера (64.188.57.249) — туннель работает
- Включили системный прокси в Windows (Параметры → Сеть → Прокси, 127.0.0.1:10808) — он был выключен
- Полная страница через прокси (`curl -x http://127.0.0.1:10808 https://www.google.com`) → 200
- Браузер заработал — подтвердил Вячеслав

### Решения
- VPN-инбаунд теперь работает на порту 443 (не 21533) — потому что TimeWeb блокирует нестандартные порты на уровне облачного файрвола, нужно открывать их вручную в панели
- Зафиксировано в knowledge/infrastructure.md
