### Сделано
- Старт сессии, загружен контекст
- Починен личный VPN-туннель (Hysteria2 через s-ui на TimeWeb 64.188.57.249): причина — мусорная запись клиента `test-trigger` (id=2, config=NULL, осталась с сессии 25 июня) ломала сборку конфига ядра, из-за чего Hysteria2-core падал в бесконечный цикл рестарта. Решение — полный релогин: `claude auth login --claudeai` в persistent-сессии (`tmux`/`systemd-run --uid=1000`, чтобы сессия не умирала при закрытии SSH), Вячеслав открыл выданную ссылку, дал код авторизации, вставили через `tmux send-keys` → токен обновлён, бот перезапущен, отвечает
- Попутно включил `loginctl enable-linger agent` на Beget — без этого фоновые процессы (tmux/screen), запущенные через `sudo -u agent` внутри root-сессии SSH, умирали при разрыве соединения
- Вячеслав подтвердил: бот отвечает, проблема закрыта

### Решения
- Тестовые записи в s-ui.db (созданные при дебаге триггеров/багов) — не оставлять в базе, они могут молча ломать сборку конфига ядра. Проверять `SELECT * FROM clients` на NULL config после экспериментов
- Если OAuth-токен Claude Code на сервере истекает и не обновляется сам — не долбить refresh endpoint повторно (быстро попадает под rate limit), сразу делать полный `claude auth login` через tmux
- Не спрашивать подтверждение на действия вне красной зоны (деньги/безопасность/необратимость) — Вячеслав явно попросил дважды за сессию
