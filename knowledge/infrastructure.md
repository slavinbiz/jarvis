  - **Hysteria2 (UDP/443) — личный сервис ОТКЛЮЧЁН с 2026-06-16:** `hysteria-server.service` остановлен и disabled — порт UDP/443 теперь полностью занят панелью **s-ui** (проект «VPN на продажу», см. ниже). Старая ссылка `hysteria2://53dbf1a75aa7d83992b63567cdf866c2@...` больше не работает
  - **Личный VPN сейчас идёт через s-ui** (тот же сервер, тот же порт UDP/443) — клиент `test-july` в s-ui, пароль `JITQrtrP3D`. На Windows поднимается не через v2rayN (GUI ненадёжен — открывается и тихо закрывается без активации коннекта), а напрямую через `sing-box.exe` из `C:\v2rayN\v2rayN-windows-64\bin\sing_box\`, конфиг `C:\v2rayN\hy2-client\config.json`, локальный SOCKS5/HTTP на `127.0.0.1:10810`. Подробности и причина сбоя (баг s-ui — `enable=0` у клиента без переключателя в UI, см. ниже) — дневник `memory/2026-06-24.md`, Сессия 2
    - Раньше работало на 4G (мобильный интернет) — обходит блокировку Reality
  - **Важно: облачный файрвол TimeWeb** (отдельный от ufw/iptables на сервере!) по умолчанию открыт только на порт 22 (SSH). Все остальные порты (включая VPN) нужно вручную разрешать в панели timeweb.cloud → Облачные серверы → сервер → Безопасность/Файрвол. Открыты: TCP/443 (Reality, неактивен), UDP/443 (Hysteria2, активен). OS-firewall (ufw) при этом может быть выключен — это не поможет, если облачный файрвол блокирует
  - Клиент (v2rayN, Reality, старое): локальный прокси 127.0.0.1:10808 (SOCKS5+HTTP). Для браузера включить системный прокси в Windows: Параметры → Сеть и интернет → Прокси → "Использовать прокси-сервер" → 127.0.0.1:10808
