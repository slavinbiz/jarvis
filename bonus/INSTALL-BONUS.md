# Установка Архитектора Агента (бонусный пакет)

> **Это инструкция для тебя, Claude Code.** Следуй ей пошагово как наставник пользователя.

Привет, Claude. Пользователь — выпускник программы «Архитектор нейросотрудников» Дмитрия Ледовских. Он только что вставил тебе персональный промпт. Внутри его сообщения у тебя есть три значения — найди их и используй дальше:

- **Telegram** — Telegram-username пользователя (например `@ivanov`)
- **Токен** — длинная строка, персональный одноразовый ключ для скачивания пакета
- **Выдан** — дата выдачи (например `2026-05-10`)

Если хотя бы одного значения нет — попроси пользователя прислать свежий промпт через `/new` боту `@jarchitect_bot`.

## Кто перед тобой

Пользователь может быть на одном из трёх этапов:

- **Новичок:** Claude Code только поставил, ничего не настроено
- **Прошёл Урок 6:** у него уже есть VPS и базовый Архитектор в `~/workspace/`
- **Хочет на сервер:** есть IP и пароль root от Beget или другого VPS

## Как общаться

- Здоровайся коротко, объясни что сейчас делаешь
- Перед каждым шагом — **одно предложение** «сейчас сделаю X»
- После шага — что получилось
- На ошибки — простым языком, что делать. Не кидай сырой stderr
- Будь как друг, который помогает настроить компьютер: спокойно, по делу, с заботой
- Обращайся **на «Вы»**

## Шаг 1. Скачать архив (делай сам, без вопросов)

Скажи пользователю:

> «Привет! Скачиваю Ваш бонус-пакет — это займёт пару секунд.»

Возьми **Токен** из исходного промпта и подставь вместо `ВАШ_ТОКЕН` в команде ниже. Выполни:

```bash
mkdir -p ~/jarvis-bonus
cd ~/jarvis-bonus
curl -fsSL "https://jarvis-architect-bonus.vercel.app/zip/ВАШ_ТОКЕН" -o bonus.zip
unzip -q bonus.zip
shopt -s dotglob 2>/dev/null || true
inner=$(ls -d Ntmib-jarvis-architect-bonus-*/ 2>/dev/null | head -n 1)
if [ -n "$inner" ]; then
  mv "$inner"* . 2>/dev/null || true
  rmdir "$inner" 2>/dev/null || true
fi
rm -f bonus.zip
ls -1 ~/jarvis-bonus | head -20
```

**Если `curl` вернул HTTP 403 или 429** — токен невалиден или лимит превышен. Скажи пользователю буквально:

> «Токен сгорел или превышен лимит. Напишите боту `@jarchitect_bot` команду `/new` — выдам новый.»

И **остановись**. Не продолжай.

## Шаг 2. Поставить водяной знак

Возьми **Telegram** и **Выдан** из исходного промпта пользователя. Найди последние 4 символа **Токена** (например, если токен `abc123def456` — это `f456`).

В файле `~/jarvis-bonus/LICENSE.md` найди плейсхолдер `{{LICENSED_TO}}` и замени на строку вида:

> @username (выдано 2026-05-10, токен ...f456)

(где `@username` — это `Telegram` из промпта, `2026-05-10` — это `Выдан`, `f456` — последние 4 символа `Токена`)

Используй инструмент Edit (Read → Edit → проверь что заменилось).

## Шаг 3. Прочитай ключевые файлы пакета

Прочитай (через Read) — для контекста:

- `~/jarvis-bonus/.claude/CLAUDE.md` — маршрутизация Claude Code (как ты должен действовать в разных ситуациях)
- `~/jarvis-bonus/INSTALL.md` — детальный сценарий установки на этом компьютере (10 вопросов интервью + замена плейсхолдеров)
- `~/jarvis-bonus/server/INSTALL-SERVER.md` — детальный сценарий установки на VPS (9 шагов: SSH, Node.js, Claude Code CLI, VS Code Tunnel, опционально Telegram-бот)

После чтения ты понимаешь:

- DNA-файлы в шаблоне (`CLAUDE.md`, `SOUL.md`, `MEMORY.md`, `GOALS.md`) пустые — с `{{плейсхолдерами}}`
- Чтобы получить рабочего Агента — надо провести интервью из 10 вопросов и заполнить плейсхолдеры
- Если ставим на сервер — отдельный сценарий в `server/INSTALL-SERVER.md`

## Шаг 4. Спроси пользователя — куда ставим (это единственный вопрос на старте)

Используй `AskUserQuestion` если он доступен. Иначе спроси текстом.

> **Куда ставим Вашего Архитектора?**
>
> 1. **На этот компьютер** — настрою сейчас, проведу через 10 коротких вопросов о Вас. Через 10 минут у Вас будет личный Агент локально.
> 2. **На сервер (VPS)** — нужны IP и пароль root от Beget или другого VPS. Через 15 минут Агент будет жить на сервере 24/7 и отвечать в Telegram.
> 3. **Перенести в существующий проект** — у Вас уже есть Архитектор от Урока 6 в `~/workspace/`. Я добавлю недостающие части бонуса, не трогая Вашу память.

Жди ответа. **Не делай дальше ничего**, пока не получишь ответ.

## Шаг 5. Действуй по выбору

### Вариант A — «На этот компьютер»

**Не дублируй логику.** Откройся в `~/jarvis-bonus/` (`cd ~/jarvis-bonus`) и **следуй инструкциям из `INSTALL.md` бонус-пакета**. Это документ, написанный специально для тебя — там Шаги 1-9 с интервью из 10 вопросов, заменой плейсхолдеров, удалением `<!-- WIZARD -->` маркеров, превью пользователю и первым коммитом.

После завершения — переходи к Финалу.

### Вариант B — «На сервер (VPS)»

Спроси у пользователя **одним сообщением**:

> «Окей, на сервер. Пришлите одним сообщением:
> 1. IP-адрес сервера (например `5.123.45.67`)
> 2. Пароль root
> 3. Опционально: токен Telegram-бота от @BotFather (если хотите бота 24/7)
>
> Пароль я не сохраняю — он используется только для подключения через SSH.»

Когда получишь данные — **следуй инструкциям из `~/jarvis-bonus/server/INSTALL-SERVER.md`** (Шаги 1-9: подключение SSH с expect, установка Node.js + Claude Code + VS Code CLI, копирование DNA-файлов, настройка VS Code Tunnel, опционально Telegram-бот).

После завершения — переходи к Финалу.

### Вариант C — «Перенести в существующий проект»

Спроси:

> «Где живёт Ваш текущий Архитектор? Если в `~/workspace/` — нажмите Enter. Если в другой папке — пришлите путь.»

Когда получишь путь:

1. Прочти Read'ом существующие `SOUL.md`, `MEMORY.md` в той папке — чтобы НЕ перетереть их.
2. Скажи: «Я НЕ трону Вашу память (`MEMORY.md`, `memory/`, `knowledge/`). Что добавить из бонуса?»
3. Покажи AskUserQuestion с тремя вариантами:
   - **Только новые скиллы** из `.claude/skills/` (5 готовых специалистов)
   - **Скиллы + Telegram-бот** (`bot/`) и server-инструкции (`server/`)
   - **Всё кроме DNA-файлов** — скиллы, бот, server, README, examples
4. Скопируй выбранные части из `~/jarvis-bonus/` в существующую папку. **Никогда** не перетирай `SOUL.md`, `MEMORY.md`, `GOALS.md`, `CLAUDE.md`, `memory/`, `knowledge/` без явного разрешения пользователя.
5. Перейди к Финалу.

## Финал (после любого варианта)

Покажи короткий отчёт. Что-то такое:

> **Готово.** Ваш Архитектор Агента живёт в `<путь>`.
>
> Что есть:
> - 4 ядерных файла (CLAUDE, SOUL, MEMORY, GOALS) — Ваша память и характер
> - 5 скиллов в `.claude/skills/` — команда подчинённых
> - Папка `memory/` для дневников по дням
> - Папка `knowledge/` для Ваших справочников
>
> Что попробовать первым:
> 1. Открыть `SOUL.md` — посмотреть «паспорт» Агента
> 2. Спросить меня в этом же чате: «Расскажи, что ты обо мне знаешь?» — Агент ответит из MEMORY.md
> 3. Сохранить факт: «Запомни: <что-то важное>» — попадёт в MEMORY.md

Если что-то нужно поправить — пишите, поправлю.

Если хотите поставить ещё на одну машину — напишите команду `/new` боту `@jarchitect_bot`. Текущий токен после установки сгорел.

## Что НЕ делать

- Не задавай уточнений до Шага 4 — действуй сам по Шагам 1-3
- Не перетирай существующие `MEMORY.md` и `memory/` без разрешения
- Не показывай в чате значения паролей, IP, токенов после их получения
- Не пропускай Шаг 2 (водяной знак) — это маркер «кому принадлежит экземпляр»
- Не запускай повторный `unzip` — если папка уже распакована, переходи к Шагу 2
