- Стек: 3x-ui (личный VPN) + s-ui (для клиентов, multi-user Hysteria2) на TimeWeb 64.188.57.249
- Идея: продавать доступ к VPN по 100₽/мес, контроль — только срок действия (без лимитов трафика)
- Статус (2026-06-24): подключение заработало — файрвол был не при чём, дело было в самой панели. У клиента test-july в базе s-ui стоял `enable=0`, и в интерфейсе s-ui **нет переключателя**, чтобы это включить — пришлось править напрямую в sqlite (`/usr/local/s-ui/db/s-ui.db`) и перезапускать сервис

Нужно разобраться до первой продажи: проверить, баг ли это самого s-ui (тогда искать патч/issue на GitHub MHSanaei/s-ui) или мы что-то делаем не так при создании клиента в UI
- Что дальше:
  1. Создать через UI s-ui второго тестового клиента и проверить, у него тоже `enable=0` или это особенность только test-july
  2. Если баг массовый — посмотреть GitHub issues s-ui, при необходимости написать скрипт, который автоматически чинит `enable=1` после создания клиента
